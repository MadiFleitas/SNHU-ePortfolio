Artifact 1: Inventory Management Android App

Description

This artifact is an Android app developed for managing inventory items. It was initially created during a mobile development course and includes features such as adding, viewing, updating, and deleting inventory records. The app was built using Kotlin and leverages a local SQLite database for data persistence, providing an offline mode for managing inventory data without requiring internet access.

Justification

I selected this artifact because it demonstrates my skills in software design and engineering. The project showcases my ability to create a functional and user-friendly application, manage data persistence, and apply software architecture principles, such as the Model-View-ViewModel (MVVM) pattern. This architecture separates the app's business logic from its user interface, making the code more maintainable and scalable. Enhancements included expanding the app's core functionality to a web-based platform for increased accessibility and a broader user base.

Technologies Used

  • Kotlin: The primary programming language used to develop the app, chosen for its modern syntax and compatibility with the Android platform.
  • SQLite: Used for local data storage, providing an embedded database solution that allows the app to perform CRUD operations on inventory items.
  • Android Studio: The integrated development environment (IDE) used for writing, testing, and debugging the app's code.
  • MVVM Architecture: The app follows the Model-View-ViewModel pattern to separate data handling from the user interface, promoting a clean code structure.

Challenges Faced

During the development of this app, one of the main challenges was ensuring data synchronization between the local SQLite database and the app's user interface, especially when multiple data updates occurred simultaneously. Additionally, implementing the MVVM architecture required careful consideration to ensure that the data flow remained consistent and that the app responded correctly to changes in the data.

Solutions Implemented

To address the synchronization issue, I used Android's LiveData and ViewModel classes to create a reactive data flow. These components allowed the app to automatically update the user interface whenever the data changed, ensuring a seamless user experience. Additionally, I used the Room library to simplify SQLite database interactions and handle data operations asynchronously, which helped avoid potential performance issues related to database access.

Enhancement Reflection

While enhancing this artifact, I learned a great deal about the complexities of cross-platform development and refined my skills in software design. The process involved restructuring the app's architecture to make it compatible with a web-based framework, which required significant adaptation of the existing codebase. This enhancement improved the app's usability by allowing it to run on both mobile and web platforms, thereby reaching a broader audience.

Specifically, the enhancement involved:

  • Implementing a RESTful API for data synchronization between the Android app and the web-based version, enabling seamless data sharing.
  • Adapting the data layer to support both SQLite for offline use and a cloud-based database for web access.
  • Modifying the user interface to ensure consistency across different platforms while maintaining platform-specific features for a native feel.

Lessons Learned

The development and enhancement of the Inventory Management Android App provided valuable insights into mobile application development and cross-platform compatibility. I gained a deeper understanding of using architecture patterns like MVVM to create maintainable code and managing data synchronization in mobile apps. Additionally, I improved my skills in designing user interfaces that work well across multiple devices.

In future projects, I plan to incorporate cloud-based services from the start to simplify data synchronization across platforms and consider using hybrid development frameworks (e.g., Flutter or React Native) for easier cross-platform development.
