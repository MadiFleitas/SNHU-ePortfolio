Overview

The Inventory Management System started as an Android app for basic inventory management and evolved into a comprehensive solution incorporating a web-based JavaScript application and a database-enhanced dashboard. Each stage focuses on different aspects, including software design, data optimization, and database integration, allowing the system to support efficient inventory management across multiple platforms.

Software Design and Engineering

The initial version was developed as an Android application using Kotlin. It supports basic CRUD operations for managing inventory items, such as adding, viewing, updating, and deleting records. The DataAdapter.kt file handles data persistence and database interaction.

The project later extended to a web-based application using JavaScript. The web app features a responsive user interface and provides cross-platform accessibility. It is built using components like MainComponent.js and OtherComponents.js, following the Model-View-Controller (MVC) architecture for better organization and maintainability.

Algorithms and Data Structures

The web-based app includes various algorithmic improvements for data handling, including caching mechanisms implemented in JavaScript to optimize data retrieval based on usage patterns. These techniques reduce the number of direct database queries, resulting in better performance. Sorting and filtering algorithms were also implemented to enable efficient data searching.

Data structures were used to validate user inputs and maintain consistent data formats. Additionally, algorithms were developed to ensure data integrity by performing checks before data was added or modified.

Databases

The system integrates a database backend that supports both the Android and JavaScript versions of the app. Queries handle data manipulation tasks such as searching, sorting, and aggregating inventory data. A web-based dashboard visualizes inventory data using database queries and supports data mining techniques to refine search results and present relevant information to the user.

For the JavaScript version, the Mongoose library was used for schema creation and database access, providing a structured approach to data operations. Controllers and routes were defined to manage API endpoints, and the database functionality was thoroughly tested using Postman to ensure robust and secure data access.

Getting Started

Prerequisites:

  • Android Studio for the Android app version
  • Node.js and npm for the JavaScript web app
  • A MongoDB database for storing inventory data
  • Git for version control

Project Structure

The project is organized into separate sections for each version of the application, with a consistent file structure for easy navigation:

  • Android App: Contains the Kotlin-based code for the mobile application, including the DataAdapter.kt file for database interactions.
  • Web-Based JavaScript App: Includes the main components, such as MainComponent.js and OtherComponents.js, following the MVC pattern.
  • Web Dashboard: Contains database integration and visualization code, with schemas, routes, and controllers managed using Mongoose.

Future Enhancements

Future improvements may include adding user authentication, expanding the caching system, and integrating predictive analytics for inventory trends. Additional performance optimizations for the web dashboard and mobile app are also under consideration.
