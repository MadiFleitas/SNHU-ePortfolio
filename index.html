Welcome to My Inventory Management System ePortfolio

This project is a comprehensive inventory management system developed in three different stages: an Android app, a web-based JavaScript app, and a database-enhanced web dashboard. The project demonstrates skills in Software Design and Engineering, Algorithms and Data Structures, and Databases, providing a complete solution for inventory management across multiple platforms.

Overview

The Inventory Management System began as an Android app, then evolved into a web-based JavaScript application, and finally integrated advanced database functionalities with a dashboard for better data presentation. Each stage addresses different aspects of the project, focusing on improving design, algorithms, and data management.

Software Design and Engineering

The initial version was developed as an Android application using Kotlin. The app supports basic CRUD operations for managing inventory items, including adding, viewing, updating, and deleting records. The DataAdapter.kt file handles data persistence and database interaction.

The project was then extended to a web-based application using JavaScript, featuring a responsive user interface and cross-platform access. It is built using components like MainComponent.js and OtherComponents.js, following the Model-View-Controller (MVC) pattern for better code organization.

Algorithms and Data Structures

The web-based app improves data handling through caching mechanisms implemented in JavaScript, optimizing data retrieval based on usage patterns. This helps reduce the number of direct database queries and improves overall performance. Algorithms for sorting and filtering inventory items have been implemented to allow efficient searching.

Data structures are used to validate user inputs and maintain consistent data formats. Additional algorithms ensure data integrity by performing checks before data is added or modified.

Databases

The project integrates a database backend that supports both the Android and JavaScript versions. Queries handle data manipulation tasks, such as searching, sorting, and aggregating inventory data. A web-based dashboard is included to visualize inventory data using queries, supporting data mining techniques to refine search results and present relevant information.

In the JavaScript version, the Mongoose library is used for schema creation and database access, providing a structured approach to data operations. Controllers and routes manage API endpoints, tested using Postman to ensure robust and secure database access.

Getting Started

Prerequisites:

  • Android Studio for the Android app version
  • Node.js and npm for the JavaScript web app
  • A MongoDB database for storing inventory data
  • Git for version control

The project is organized into separate sections for each version of the application, with a consistent file structure for easy navigation.
